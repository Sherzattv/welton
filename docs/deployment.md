# Деплой

Сайт хостится на **GitHub Pages**: https://sherzattv.github.io/welton/

Источник — ветка **`gh-pages`**, в которой лежит собранный сайт (содержимое
`dist/`). Исходный код — в ветке `main`.

## Как обновить опубликованный сайт

После того как правки в `main` готовы и проверены локально (`npm run dev`),
выполните:

```bash
# 1. Собрать сайт
npm run build

# 2. Обновить ветку gh-pages собранным dist/
cd /tmp
rm -rf welton-ghp
git clone --branch gh-pages --single-branch https://github.com/Sherzattv/welton.git welton-ghp
cd welton-ghp
git rm -rq .                              # убрать старую сборку
cp -R /путь/к/проекту/welton/dist/. .     # положить новую
touch .nojekyll                           # обязательно (см. ниже)
git add -A
git commit -m "Redeploy"
git push origin gh-pages
```

GitHub соберёт Pages из обновлённой ветки за 1–2 минуты, и сайт обновится по
тому же адресу. Контент `main` при этом не меняется — туда коммитьте исходники
отдельно.

> **Зачем `.nojekyll`:** GitHub Pages по умолчанию прогоняет контент через Jekyll,
> а тот игнорирует папки, начинающиеся с `_` (у нас это `_astro/` со всеми
> стилями и скриптами). Пустой файл `.nojekyll` отключает Jekyll — без него сайт
> поедет без стилей.

## Конфигурация Pages

- `astro.config.mjs` задаёт `site` и `base: '/welton'` — потому что проект
  публикуется под под-путём `/welton/`. Не убирайте `base`, иначе ссылки на
  стили и картинки сломаются.
- Источник Pages (ветка `gh-pages`) настроен в репозитории: **Settings → Pages**.

## Автодеплой через GitHub Actions (сейчас отключён)

В репозитории есть готовый workflow
[`.github/workflows/deploy.yml`](../.github/workflows/deploy.yml): он собирает
сайт и публикует на Pages при каждом пуше в `main` — то есть ручные шаги выше
становятся не нужны.

**Но сейчас он не работает:** аккаунт GitHub заблокирован по биллингу, поэтому
любые Actions падают на старте с ошибкой
_«account is locked due to a billing issue»_. Пока это так, обновлять сайт нужно
вручную (раздел выше).

Чтобы включить автодеплой:

1. Снять блокировку в **GitHub → Settings → Billing** (погасить задолженность /
   обновить способ оплаты / поднять лимит трат).
2. В **Settings → Pages** переключить источник на **GitHub Actions**.
3. После этого каждый пуш в `main` будет публиковаться автоматически; ветка
   `gh-pages` больше не понадобится.

## Требования к репозиторию

GitHub Pages на бесплатном тарифе работает только для **публичных** репозиториев.
`Sherzattv/welton` уже публичный — менять ничего не нужно.
