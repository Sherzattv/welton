# Как менять контент

Почти всё наполнение главной страницы лежит в одном файле —
[`src/data/home.ts`](../src/data/home.ts). Тексты, контакты и пути к изображениям
правятся там, разметку трогать не нужно.

После любой правки проверьте результат локально:

```bash
npm run dev      # http://localhost:4321/welton/
```

А когда всё устраивает — опубликуйте (см. [deployment.md](deployment.md)).

---

## Где что лежит в `home.ts`

Файл — это набор именованных блоков (экспортов). Каждый соответствует секции:

| Экспорт | Секция на сайте | Что содержит |
|---|---|---|
| `contact` | шапка, футер, заявка | телефон, email, адрес, мессенджеры, реквизиты |
| `nav` | меню | пункты навигации |
| `stats` | счётчики | цифры и подписи |
| `services` | направления | 3 карточки: номер, заголовок, описание, картинка |
| `advantages` | преимущества | 4 пункта: заголовок + описание |
| `objects` | объекты | карточки слайдера: заголовок, описание, картинка |
| `partners` | партнёры | название + логотип |
| `stages` | этапы работы | номер, заголовок, описание |
| `faq` | вопросы-ответы | вопрос + ответ |

## Поменять текст

Найдите нужный блок и отредактируйте строку в кавычках. Пример — заголовок услуги:

```ts
export const services = [
  {
    num: "01",
    title: "ПВХ-остекление объектов",          // ← правьте здесь
    desc: "Реализуем ПВХ-остекление для…",      // ← и здесь
    img: "/img/124_2.jpg",
  },
  // …
];
```

Правила:
- текст всегда внутри `"кавычек"`;
- каждый элемент списка отделяется запятой;
- не удаляйте фигурные `{ }` и квадратные `[ ]` скобки — это структура.

## Поменять контакты

Все контакты — в блоке `contact` сверху файла. Меняется и видимый текст, и ссылка:

```ts
export const contact = {
  phone: "+7 903 021 12 88",          // как показывается
  phoneHref: "tel:+79030211288",      // куда ведёт клик (без пробелов!)
  email: "zakaz.velesgroup@mail.ru",
  emailHref: "mailto:zakaz.velesgroup@mail.ru",
  address: "107564, г.Москва, …",
  telegram: "https://t.me/velesgroup2026",
  // …
};
```

## Поменять или добавить изображение

1. Положите файл в [`public/img/`](../public/img) (для видео — `public/video/`).
2. В `home.ts` укажите путь от корня `public`, начиная со слэша:

```ts
img: "/img/мой-новый-файл.jpg",
```

> Пути к картинкам в данных пишутся как `/img/...`. Префикс `/welton/` не нужен —
> он добавляется автоматически при рендере. Подробнее — в
> [architecture.md](architecture.md#пути-к-статике-и-base-путь).

Рекомендации по файлам: имена латиницей без пробелов; фото — `.jpg`, логотипы и
иконки — `.svg`/`.png`; крупные фото перед загрузкой имеет смысл сжать.

## Добавить или убрать карточку

Карточки услуг, объектов, преимуществ, этапов — это элементы списка. Чтобы
добавить — скопируйте существующий блок `{ … }` и вставьте рядом через запятую:

```ts
export const stages = [
  { num: "01", title: "Анализ объекта", desc: "…" },
  { num: "02", title: "Расчёт", desc: "…" },
  { num: "03", title: "Новый этап", desc: "…" },   // ← добавили
];
```

Чтобы убрать — удалите соответствующий блок `{ … }` целиком (вместе с запятой).
Нумерацию (`num`) поправьте вручную, если она идёт по порядку.

## Поменять заголовки секций и подписи

Часть подписей (например, заголовок «Объекты, с которыми мы работаем») задана
прямо в разметке секции, а не в `home.ts`. Такие тексты лежат в
[`src/components/sections/`](../src/components/sections) — по файлу на секцию.
Откройте нужный (`Objects.astro`, `Services.astro` и т.д.) и отредактируйте текст
между тегами. Структуру и классы не трогайте.

## Добавить новую секцию (для разработчика)

1. Создайте `src/components/sections/Новая.astro` — за образец возьмите соседнюю
   секцию (контейнер, `<GridLines />`, scoped-стили).
2. Данные секции добавьте отдельным экспортом в `home.ts`.
3. Подключите секцию в [`src/pages/index.astro`](../src/pages/index.astro) в нужном
   месте порядка.
4. Переиспользуйте `Heading`, `Button`, токены и motion-паттерны (`data-reveal`,
   `data-split`) — см. [architecture.md](architecture.md).

## Если что-то сломалось

- `npm run build` показывает ошибку — чаще всего это пропущенная запятая или
  кавычка в `home.ts`. Проверьте последнюю правку.
- Изменения не видны — перезагрузите страницу; если не помогло, перезапустите
  `npm run dev`.
- Любую правку можно откатить через git (история сохраняется).
