# Как менять контент

Контент главной — в [`src/data/home.ts`](../src/data/home.ts). Сквозные данные
(контакты, навигация) — в [`src/data/site.ts`](../src/data/site.ts). Каталог
проектов — в Content Collection и `projectOverrides.ts` (см. ниже). Тексты и
пути к изображениям правятся в этих файлах, разметку трогать не нужно.

После любой правки проверьте результат локально:

```bash
npm run dev      # http://localhost:4321/
```

А когда всё устраивает — опубликуйте (см. [deployment.md](deployment.md)).

---

## Где что лежит

| Файл                                                              | Что содержит                                                                               |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| [`src/data/site.ts`](../src/data/site.ts)                         | `contact`, `nav` — сквозные данные шапки, футера и блока заявки                            |
| [`src/data/home.ts`](../src/data/home.ts)                         | контент главной: `stats`, `services`, `advantages`, `objects`, `partners`, `stages`, `faq` |
| [`src/data/projectPage.ts`](../src/data/projectPage.ts)           | сквозные блоки детальной страницы проекта (`fullCycle`, `customization`)                   |
| [`src/data/projectOverrides.ts`](../src/data/projectOverrides.ts) | ручные надписи к проектам: `title`, `tagline`, категория, технология…                      |
| `public/projects/NN/project.json`                                 | геометрия конкретного проекта: `area`, `width`, `depth`, `floors`                          |

Схема данных типизирована в [`src/types.ts`](../src/types.ts) и валидируется
Zod-схемой коллекции `projects` в
[`src/content.config.ts`](../src/content.config.ts) — опечатка ловится на сборке.

## Поменять текст

Найдите нужный блок и отредактируйте строку в кавычках. Пример — заголовок
услуги в `home.ts`:

```ts
export const services = [
  {
    num: "01",
    title: "Монолитные дома", // ← правьте здесь
    desc: "Монолитно-каркасная …", // ← и здесь
  },
  // …
];
```

Правила:

- текст всегда внутри `"кавычек"`;
- каждый элемент списка отделяется запятой;
- не удаляйте фигурные `{ }` и квадратные `[ ]` скобки — это структура.

## Поменять контакты

Все контакты — в блоке `contact` в [`src/data/site.ts`](../src/data/site.ts).
Меняется и видимый текст, и ссылка:

```ts
export const contact = {
  phone: "+7 700 510 10 30", // как показывается
  phoneHref: "tel:+77005101030", // куда ведёт клик (без пробелов!)
  email: "mail@welton.kz",
  emailHref: "mailto:mail@welton.kz",
  address: "г. Алматы, ул. Кунаева, 43",
  instagram: "https://instagram.com/welton.kz",
  company: "ТОО «Велтон»",
} satisfies ContactInfo;
```

## Поменять или добавить изображение

1. Положите файл в [`public/img/`](../public/img) (видео — `public/video/`,
   изображения проекта — `public/projects/NN/`).
2. В `home.ts` укажите путь от корня `public`, начиная со слэша:

```ts
img: "/img/мой-новый-файл.jpg",
```

> Префикс домена не нужен — он добавляется автоматически хелпером `asset()` при
> рендере. Подробнее — в [architecture.md](architecture.md#пути-к-статике).

Рекомендации по файлам: имена латиницей без пробелов; фото — `.jpg`, логотипы и
иконки — `.svg`/`.png`; крупные фото перед загрузкой имеет смысл сжать.

## Добавить или убрать карточку

Карточки услуг, объектов, преимуществ, этапов — это элементы списка. Чтобы
добавить — скопируйте существующий блок `{ … }` и вставьте рядом через запятую:

```ts
export const stages = [
  { num: "01", title: "Анализ объекта", desc: "…" },
  { num: "02", title: "Расчёт", desc: "…" },
  { num: "03", title: "Новый этап", desc: "…" }, // ← добавили
];
```

Чтобы убрать — удалите соответствующий блок `{ … }` целиком (вместе с запятой).
Нумерацию (`num`) поправьте вручную, если она идёт по порядку.

## Поменять заголовки секций и подписи

Часть подписей (например, заголовок «Объекты, с которыми мы работаем») задана
прямо в разметке секции, а не в `home.ts`. Такие тексты лежат в
[`src/components/sections/`](../src/components/sections) — по файлу на секцию.
Откройте нужный (`Objects.astro`, `Services.astro` и т. д.) и отредактируйте
текст между тегами. Структуру и классы не трогайте.

## Каталог проектов

Каталог — это ~80 объектов в `public/projects/NN/`. У каждой папки своя
геометрия и набор изображений; Astro собирает их в коллекцию `projects` на
билде (см. [architecture.md → Каталог проектов](architecture.md#каталог-проектов)).

### Поменять надписи у конкретного проекта

`title`, `tagline`, категория, технология, число спален и т. д. — в
[`src/data/projectOverrides.ts`](../src/data/projectOverrides.ts). Маппинг по
id записи коллекции (`wlt-NN`):

```ts
export const projectOverrides = {
  "wlt-03": {
    title: "Двусвет",
    tagline: "Барнхаус с панорамным остеклением и террасой на юг.",
    category: "barnhouse",
    bedrooms: 3,
    bathrooms: 1,
  },
  // …
};
```

Незаданные проекты остаются с placeholder-именем «WLT-NN» без tagline.

### Поменять сквозные блоки детальной страницы

«Полный цикл работ» и «Дополнения в проект» — общие для всех проектов и лежат в
[`src/data/projectPage.ts`](../src/data/projectPage.ts). Меняются один раз —
обновляются у всех ~80 проектов.

### Поменять геометрию или изображения проекта

- Геометрия (`area`, `width`, `depth`, `floors`, `pages`, `note`) —
  `public/projects/NN/project.json`.
- Фасад — `public/projects/NN/facade.png`.
- 3D-визуализации — `3d-1.jpg`, `3d-2.jpg` (необязательно обе).
- Планировки — `plan.png` или `plan-1.png` / `plan-2.png`.

Loader сам подхватит, какие файлы реально лежат в папке.

## Добавить новую секцию (для разработчика)

1. Создайте `src/components/sections/Новая.astro` — за образец возьмите соседнюю
   секцию (контейнер, `<GridLines />`, scoped-стили).
2. Данные секции добавьте отдельным экспортом в `home.ts` (типизируйте через
   `satisfies` и интерфейс из [`src/types.ts`](../src/types.ts)).
3. Подключите секцию в [`src/pages/index.astro`](../src/pages/index.astro) в нужном
   месте порядка.
4. Переиспользуйте `Heading`, `Button`, `ui/`-примитивы, токены и motion-паттерны
   (`data-reveal`, `data-split`); импорты — через алиасы. Ширину обёртки задавайте
   через `<Container>` или `max-w-[var(--content-max)] px-gutter`, а не хардкодом.
   См. [architecture.md](architecture.md).

## Если что-то сломалось

- `npm run check` / `npm run build` показывает ошибку — чаще всего пропущенная
  запятая или кавычка в данных. Проверьте последнюю правку.
- Изменения не видны — перезагрузите страницу; если не помогло, перезапустите
  `npm run dev`.
- Любую правку можно откатить через git (история сохраняется).
