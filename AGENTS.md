# Соглашения для разработчиков и AI-агентов

Краткие правила работы над проектом. Полные детали — в [README.md](README.md) и
[docs/](docs).

## Что это

Welton — статический сайт-одностраничник на Astro 7 + Tailwind CSS 4 (TypeScript,
strict). Масштабируемая архитектура: переиспользуемые секции, контент в данных,
единые дизайн-токены. Никакого фреймворк-рантайма на клиенте, кроме motion-слоя.

## Главные правила

- **Контент — в данных, не в разметке.** Контент главной — в
  [`src/data/home.ts`](src/data/home.ts); сквозные контакты/навигация — в
  [`src/data/site.ts`](src/data/site.ts). Формы данных типизированы в
  [`src/types.ts`](src/types.ts) через `satisfies`. См.
  [docs/content-editing.md](docs/content-editing.md).
- **Оформление — в токенах.** Цвета, типографику, отступы, ширину контента
  (`--content-max`), брейкпоинты меняйте в `@theme`/`:root` внутри
  [`src/styles/global.css`](src/styles/global.css), а не локально по секциям.
- **Переиспользуйте компоненты и motion-паттерны.** Собирайте из `Heading`,
  `Button`, `Container`, `GridLines` и примитивов `src/components/ui/`
  (`Icon`, `Hairline`, `SlideLink`); для анимаций — `data-reveal` / `data-split`,
  а не одноразовые скрипты. Импорты — через алиасы (`@components`, `@ui`, `@data`…).
  См. [docs/architecture.md](docs/architecture.md).
- **Статика — через `asset()`.** Пути к файлам из `public/` оборачивайте в
  [`asset()`](src/lib/asset.ts). Сейчас сайт обслуживается из корня домена
  (`base` не задан), но `asset()` оставляем как единый способ строить пути — он
  переживёт смену `base` и не даст «голым» `/img/...` разойтись по проекту.
- **Сетка-направляющих — единый источник.** Координаты `--guide-a/b/c` в
  `global.css`; в секции добавляйте `<GridLines />`, не задавая координаты заново.

## Рабочий процесс

- `main` — рабочая ветка с исходниками. Дев-сервер: `npm run dev` →
  `http://localhost:4321/`.
- Перед коммитом: `npm run check && npm run lint && npm run format:check` (типы,
  линт, формат) и `npm run build`. Те же проверки гоняет CI-гейт `quality` перед
  деплоем. Визуальную часть проверяйте глазами в браузере.
- Деплой — автоматический: Cloudflare пересобирает и публикует сайт при каждом
  пуше в `main`. См. [docs/deployment.md](docs/deployment.md).

## Чего не делать

- Не писать абсолютные `/img/...`, `/video/...` в обход `asset()`.
- Не хардкодить `max-w-[1680px]` — только `max-w-[var(--content-max)]` или `<Container>` (CI-гейт блокирует).
- Не дублировать `--s` по секциям — он один в `:root`.
- Не дублировать значения цветов/размеров вместо токенов.
- Не писать относительные `../../`-импорты — используйте алиасы (`@components`, `@ui`, `@data`…).
- Не задавать `base` в `astro.config.mjs` — сайт живёт на своём домене из корня.
- Не копировать чужую разметку — собирать из своих компонентов и `ui/`-примитивов.
